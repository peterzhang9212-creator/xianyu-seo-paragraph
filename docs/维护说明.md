# 维护说明

## 1. 目录结构

- `app.py`
  程序入口。
- `config/defaults.json`
  保存上一次输入值和默认参数。
- `core/assembly.py`
  文件组装逻辑。
- `core/watermark.py`
  水印批处理包装层。
- `core/content_import.py`
  正文模板读取和校验。
- `core/upload_sheet.py`
  上传表生成逻辑。
- `core/subfolder_export.py`
  独立的子文件夹名称导出逻辑。
- `core/publish_runner.py`
  发布步骤统一入口。
- `ui/main_window.py`
  主窗口和状态卡片。
- `ui/steps/`
  五个步骤/工具页的界面和交互。
- `vendor/batch_watermark_agent.py`
  迁入的水印引擎。
- `vendor/xianyu_uploader/`
  迁入的闲鱼自动发布核心。
- `templates/content_template.xlsx`
  正文导入模板。

## 2. 常改位置

### 改默认参数

- `config/defaults.json`
- `core/models.py`

### 改文件组装规则

- `core/assembly.py`

常见调整点：

- 文件夹命名规则
- 主图固定文件名
- 标题文本生成逻辑
- 封面图和详情图复制规则

### 改水印规则

- `core/watermark.py`
- `vendor/batch_watermark_agent.py`

常见调整点：

- 输出目录规则，例如 `原目录名_水印`
- 水印大小、密度、透明度
- 支持的图片格式
- 同尺寸图片的缓存策略

### 改正文模板列

- `core/content_import.py`
- `templates/content_template.xlsx`

当前必填列：

- `导出商品名`
- `标题+正文+标签`
- `发货属地`

### 改上传表结构

- `core/upload_sheet.py`

常见调整点：

- 表头名称
- 输出工作表结构
- 正文与素材的匹配规则

### 改子文件夹导出工具

- `core/subfolder_export.py`
- `ui/steps/step_subfolder_export.py`

### 改发布默认规格和价格

- `config/defaults.json`
- `core/models.py`
- `ui/steps/step_publish.py`

### 改发布底层逻辑

- `core/publish_runner.py`
- `vendor/xianyu_uploader/`

如果闲鱼页面改版，优先检查：

- `vendor/xianyu_uploader/publisher.py`
- `vendor/xianyu_uploader/bitbrowser.py`

## 3. 自动带入逻辑

状态保存集中在：

- `core/project_state.py`

当前关键状态字段：

- `last_assembly_output`
- `last_watermark_output`
- `last_upload_sheet`
- `last_content_path`

如果想改默认带入顺序，优先看：

- `ui/steps/step_watermark.py`
- `ui/steps/step_upload_sheet.py`
- `ui/steps/step_publish.py`

## 4. 自包含原则

这个项目已经把运行时需要的核心逻辑放进了本目录：

- 水印逻辑来自 `vendor/batch_watermark_agent.py`
- 闲鱼发布逻辑来自 `vendor/xianyu_uploader/`

后续维护时，不要再从旧目录直接 `import` 脚本。  
如果要升级旧逻辑，应该把需要的代码复制进本项目，再在本项目里改。

## 5. 测试命令

在项目目录运行：

```powershell
python -m pytest tests -v
```

## 6. 手工验证建议

### 验证组装

- 用 1 条城市数据测试
- 检查是否生成 `商品主图.jpg` 和 `商品标题.txt`

### 验证水印

- 用 1 个商品文件夹测试
- 检查原图未覆盖
- 检查独立输出目录是否生成

### 验证上传表

- 用 1 条正文数据测试
- 检查生成表中 `文件位置` 是否正确

### 验证发布

- 先跑 `执行预检`
- 再确认 `BitBrowser`、闲鱼登录态、上传 Excel 都正常后再点 `开始发布`
